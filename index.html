How it works

Obviously this POC is rough and ready, and the teaching is terrible(!) but the general idea is that we build a 'dummy' website featuring all the concepts we want to teach, then wrap it with tasks .

  • An iframe contains the standalone 'dummy' website that is just a basic HTML clickthrough with a tiny bit of control Javascript.
  • Importantly because it is just HTML it can be made fully branded, fully responsive, fully accessible and easily translatable.
  • All the instruction sits outside of the 'dummy' website.
  • The outer frame contains all the instructions for the task and any feedback.
  • All instructions and fields to target are defined in a simple config file. Example file
  • Different tasks can use the same app but target different fields.
  • There are different messages for each step and different messages if you do the wrong thing.
  • You can toggle highlighting of the fields you need to fill to help
  • Note: To get the next instruction you currently need to click or tab out of the field

Other things we could do fairly easily with this approach:

  • Make fields either fuzzy matches or allowing anything that matches the validation
  • Saving any data a learner types to the LRS
  • Put in place a framework to easily screenshot parts of the 'dummy' app for use in learning objects
