How it works

  • IFrame contains a standalone 'app' that is just a basic HTML clickthrough with a tiny bit of control Javascript.
  • Importantly because it is just HTML it can be made fully branded, fully responsive, fully accessible and easily translatable.
  • All the control sits outside of the 'app'.
  • The outer frame contains all the instructions and any feedback.
  • All instructions are created via a simple config file. Example file
  • There are different messages for each step if you do the wrong thing.
  • You can toggle highlighting of each
  • Note: To get the next instruction you currently need to click or tab out of the field

Other things we could do with this approach:

  • Make fields either fuzzy matches or allowing anything that matches the validation
  • Saving to the LRS anything a learner types in
  • Put in place tech to easily screenshot all this
